# Reflexión sobre git blame

## ¿Para qué usar git blame en un equipo?
En un equipo, usar git blame es útil para depurar bugs, entender quién cambió qué en el código y saber a quién preguntar sobre decisiones de código.

## ¿Por qué es mala idea usar blame para “señalar culpables”?
Usar git blame para señalar culpables crea un ambiente tóxico. Debe usarse de manera constructiva para entender el contexto de los cambios y no para señalar a los desarrolladores.

## Buenas prácticas para un blame útil y legible
1. Usar un formato consistente para los mensajes de commit.
2. Hacer commits pequeños y enfocados.
3. Evitar commits masivos de re-formateo.
